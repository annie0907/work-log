🎯 方案 A：使用 Cursor 的完整流程✅ 每月 1 號要做的事（約 5 分鐘）

## 第 1 步：開啟 Cursor 並打開專案
開啟 Cursor
File → Open Folder
選擇你的專案資料夾：

   C:\Users\ga0897\Downloads\個人物品\工作日誌更新\work-log


## 第 2 步：同步最新版本
開啟終端機（按 `Ctrl + ``）
輸入以下指令：

cd work-log
git pull origin main

看到「Already up to date」或「Updating...」就對了


## 第 3 步：開啟 Cursor Chat

按 Ctrl + L 開啟 Chat 面板
Chat 面板會出現在右側或底部


## 第 4 步（自動）：產生「上個月」Prompt 並貼上

在專案根目錄開啟終端機，執行：

powershell -ExecutionPolicy Bypass -File scripts/last-month-prompt.ps1

完成後，Prompt 已複製到剪貼簿，直接在 Cursor Chat 貼上即可。

（若無法使用腳本，改用下一節的手動方式）

## 第 4 步（手動備用）：貼上 Prompt 並送出

複製以下 Prompt（把 {{YYYY}} 與 {{MM}} 改為目標年月，MM 為兩位數）：

{{YYYY}}為2025，{{MM}}為11。

請分析 {{YYYY}}/{{MM}}/ 資料夾中所有的 .md 檔案，生成一份 {{YYYY}} 年 {{MM}} 月的詳細工作摘要。

要求：
1. 仔細閱讀所有 markdown 檔案的內容
2. 按照以下結構生成繁體中文摘要：

# {{YYYY}} 年 {{MM}} 月 工作摘要

## 📊 本月概況
（用 2-3 段話總結本月的整體工作情況、主要焦點和工作節奏）

## ✅ 主要完成事項
（列出 5-10 項最重要的完成任務，用 bullet points，每項要簡短清晰，並每個 bullet point 中間需間隔一行）

## 🎯 重要進展與成果
（突出本月的關鍵成就、里程碑或突破，說明其重要性）

## 🤔 遇到的挑戰
（整理本月遇到的主要問題、障礙或困難，以及如何應對）

## 📚 學習與成長
（總結本月學到的新技能、新知識、經驗教訓）

## 🔮 下月重點目標
（根據「明日待辦」中的內容，提取出下個月應該優先處理的事項，並每個 bullet point 中間需間隔一行）

## 💡 其他觀察與建議
（其他值得記錄的觀察、趨勢或改進建議）

3. 請將生成的摘要直接寫入新檔案：{{YYYY}}/{{MM}}/{{YYYY}}-{{MM}}-summary.md
4. 內容要詳細、有洞察力，不只是簡單列點
5. 在文件最後加上：

---
*🤖 由 AI 自動生成 *

請立即開始分析並生成摘要。貼到 Chat 面板，按 Enter 送出

第 5 步：等待 Cursor 完成
⏱️ 通常需要 30-90 秒
📝 Cursor 會自動讀取所有日誌檔案
💾 自動生成並儲存摘要檔案

## 第 6 步：檢查生成的檔案
在 Cursor 左側檔案樹，找到：

   {{YYYY}}/{{MM}}/{{YYYY}}-{{MM}}-summary.md

2. **點開檢查內容**
3. **如果不滿意，可以在 Chat 中要求修改：**

   請讓「本月概況」更詳細一些
   
## 第 7 步：上傳到 GitHub
在終端機（`Ctrl + ``）輸入：

git add {{YYYY}}/{{MM}}/{{YYYY}}-{{MM}}-summary.md
git commit -m "📊 Add monthly summary for {{YYYY}}-{{MM}}"
git push origin main
