# 2026 年 04 月 工作摘要

## 📊 本月概況

四月份整體工作節奏緊湊，以風險管理委員會（風管會）簡報為月中最大里程碑，前半月大量時間投入在資料整備與簡報製作，後半月則轉向 SOP 文件撰寫、自動化工具探索以及專案規劃推進。本月共有六個工作假日分散在週五與假日連假，工作日約 18 天，整體工作量在例行業務與專案開發之間取得平衡。

在例行業務方面，信用資訊寄發（中華徵信所、鄧白氏）、CRM 系統異常處理、報帳等事務持續穩定執行。與此同時，本月積極嘗試多種自動化工具，包括 Power Automate、Copilot、Python 排程，最終以 Python 為主軸建立了每日退補票查詢的自動化流程，展現出從工具探索到實際落地的完整過程。

專案端以分群模型和智慧客服機器人為主要推進方向，雖受限於資訊部門協作速度及系統限制，進度有所遲滯，但已逐漸明確技術路線並在月底形成下月週會討論的策略方向。

## 🎯 重要進展與成果

**風管會順利召開（04-15）**：這是本月最重要的里程碑。簡報準備歷時約兩週，涵蓋市場風險數據更新、各頁籤整理，最終在會議中順利報告。主管更於會中提出 AI 工具導入計畫及部門三年規劃，展示出部門積極推動數位轉型的企圖心。

**Power Automate 機器人雛型完成（04-28~04-29）**：雖然 Power Automate 在公司環境中有許多權限限制，仍成功建立可雙向對話的自動化機器人雛型。學長認可此方向，預計下月在週會中討論正式提案策略，有望以「競賽名義」爭取資訊部門配合開放所需資源，是未來智慧客服機器人的重要起點。

**每日退補票 Python 自動化落地（04-30）**：放棄 Copilot（抓信效果差）轉為 Python 讀取每日 Excel 產出，並設定每日排程，確保資料自動更新、避免重複查詢。雖然流程看似簡單，但從工具評估到最終落地展現了良好的問題解決能力。

**技術路線決策：Tableau → HTML/Python**：月中評估到學長的儀表板在 Tableau 上因資料權限問題嚴重卡頓，初步決定將分群模型專案改以 Python 架設 HTML 網站方式呈現，既能解決效能問題，也更符合個人技術棧，是重要的技術架構轉向。

## 🤔 遇到的挑戰

**資訊部門協作困難**：資訊需求單被踢皮球，長期找不到窗口；系統的欄位對應、ERP 整合等溝通耗費大量心力。防火牆問題也導致 CCIS 傳輸受阻，後來確認是對方防火牆封鎖。這些外部依賴是推進速度最大的瓶頸。

**CRM 系統（金腦）審查人員斷層**：原審查人員離職，其直屬主管也全數離職，需花費額外時間在組織架構中尋找合適的接手人員，增加行政流程複雜度。

**Copilot 抓信效果不佳**：原計畫用 Copilot 抓 Outlook 信件進行自動化，但連基本主旨關鍵字篩選都無法準確執行，最終放棄改用 Python，增加了開發摸索時間。

**Power Automate 系統限制多**：公司環境對 Power Automate 東鎖西鎖，許多功能受限，導致優化工作難以推進，需要靠資訊部門配合解鎖才能完整發揮。

**週會進度與溝通落差**：主管對分群模型專案估時一年覺得太長，但說明不清楚就直接被打斷；roadmap 在週會中因時間不足，幾乎沒有獲得實質回饋，專案方向的溝通效率有待加強。

## 📚 學習與成長

**Python 自動化實戰**：本月從零到一建立每日退補票 Python 排程腳本，涵蓋讀取 Excel、設定排程、避免重複查詢等實務技能，是將 Python 應用於日常工作自動化的具體成果。

**Power Automate 初探**：雖然受公司環境限制，但仍玩出了一個有來有往的機器人雛型，理解了 Power Automate 的觸發機制與回傳說明功能，為後續客服機器人方向打下基礎。

**Copilot 與 AI 工具評估能力**：出席 Copilot 訓練並實際測試，對其限制有第一手認識。同時在 Claude、Codex、Copilot 等工具間累積了比較經驗，開始形成對各工具適用場景的判斷力。

**SOP 撰寫與知識管理**：本月密集撰寫金腦 SOP、大陸風模 SOP、每日退補票 Notion 文件，深刻體會到「邊操作邊記錄」雖然效率較低，但能確實記下操作中遇到的坑洞，是知識管理的重要習慣。

**技術選型思維的建立**：在儀表板技術選型（Tableau vs HTML/Python）以及資料擷取工具選型（Copilot vs Python）上，都經歷了評估、嘗試、決策的完整過程，逐漸建立以實際需求為導向的技術判斷框架。

## 💡 其他觀察與建議

**AI 工具導入需要有人帶頭推動**：月中風管會後，高層明確表示 AI 工具引進需要有「帶頭人」且歸屬策略部門主導。這意味著部門若要實質推動 AI 工具落地，需要更主動地與策略部門對齊，或在內部累積足夠的 demo 成果來爭取支持，而非等待上層指示。

**自動化工具的選擇應以「能真正用起來」為優先**：本月嘗試了多種工具，最終發現 Python 在公司環境中反而是最不受限制、最能穩定落地的選項。未來在工具評估時，公司環境的權限限制是必須優先確認的前提條件，避免浪費時間在無法推進的方向上。

**例行事務的文件化有助於交接與稽核**：本月密集產出 SOP 文件，雖然撰寫過程枯燥，但在整理過程中也發現不少流程中的隱性知識（如寄信前要先確認簽名檔、寄完要電話追蹤特定人員等）。建議持續推進，將例行業務逐步文件化，降低未來交接風險。

**組織異動對業務連續性的衝擊不容忽視**：本月 CRM 審查人員整鏈離職的狀況，以及永續部門的內部衝突，都反映出人員流動對業務執行的潛在影響。建議在關鍵流程上確保有備份聯絡人，並在 SOP 中納入異常處理步驟。
