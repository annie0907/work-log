# 2026 年 05 月 工作摘要

## 📊 本月概況

五月是工作量相當密集的一個月，工作節奏呈現「多線並進、資訊過載」的特徵。月初主要聚焦於例行業務的整理（CCIS、雙週名單、CRM）以及 Power Platform 工具的評估；月中迎來重要轉捩點：成功將 Claude Code 與專案資料夾及 GitHub 整合，正式開啟了 AI 輔助開發的工作模式；月底則集中於審查 MBO 資料建置，以及做出終止鄧白氏合作的重大決策。

整體而言，本月在技術工具的探索與採用上有顯著進展，特別是 Claude Code 的引入改變了專案開發方式。然而也因為同時推進多個不確定性高的專案（資料庫 ER Model、Power Automate、提案書），加上外部因素如鄧白氏漲價談判破局，整體感受到的疲憊感明顯。

工作節奏上，週一至週四較為充實，週五與節假日的緩衝讓人得以喘息。全月共有工作記錄的日子 21 天，但仍有數天以「嗷嗷」（無明確產出）帶過，顯示高強度工作日與低能量日交替出現。

- **Claude Code 整合與 GitHub 連接**（5/8）：成功將 Claude Code 與專案資料夾連接並串上 GitHub，為後續 AI 輔助開發奠基，每日自動更新

- **AI/Copilot 提案書完成送出**（5/13–5/15）：從草稿、修改圖示（改 Claude 為 Copilot 圖示）到字少精煉版，歷經多次修改後正式寄出給相關人員

- **雙週名單按時寄出**（5/4、5/18）：完成兩次例行雙週名單作業

- **終止鄧白氏合作決策**（5/20、5/29）：在漲價談判破局後果斷決定停止合作，並完成通知資訊部門的後續動作

- **審查 MBO 組織檔完成**（5/26–5/29）：從問題排查到找出遺漏的篩選條件，最終完成審查人員組織檔的建置

- **新企金系統教育訓練參與**（5/21–5/22）：參加系統說明會並進行系統比對測試

- **申請門禁刷臉權限**（5/28）：解決長期無法刷臉進出大會議室的問題

## 🎯 重要進展與成果

**Claude Code 正式融入工作流程**是本月最具里程碑意義的進展。5/8 將其與專案資料夾及 GitHub 連接後，後續幾乎每一個技術任務都有 Claude Code 的參與。這不只是工具採用，而是工作方式的根本轉變——從獨自摸索改為「人機協作」模式，也讓複雜的 ER Model 設計、Schema 調整、資料比對等工作得以在單人力量下推進。

**提案書的完成與送出**是另一重要成果。從 Power 系列評估到整合 Copilot 工具的完整提案，歷經週會報告、圖示修改、格式精簡，最終成功送出並獲得每位收件者回信，代表提案的能見度與完整度達到一定水準。

**終止鄧白氏合作並規劃替代方案**也是關鍵決策。雖然這意味著先前部分自動化工作的心血付諸流水，但能在評估後做出明確決定、轉向中華徵信所方案，展現了務實的判斷力，避免繼續消耗資源在一個不划算的合作上。

## 🤔 遇到的挑戰

**Claude Code 的管控難題**貫穿整個月。Claude 傾向於認為任務已完成，但實際上往往還差一截，需要不斷「把他拉回來」。同時在 VS Code 與 Cowork 分別開啟 Claude 實例，造成注意力分散與管理混亂（5/14）。如何讓 AI 工具真正「在控制下輔助」而非「帶著走」，是本月最大的操作挑戰。

**資料庫操作技術瓶頸**明顯，5/19 資料匯入後發現問題重新來過，5/27 遭遇兩天查出資料量不同的邪教現象，5/29 自述「不會在資料庫裡下指令」。資料庫 DDL/DML 能力是需要補強的技術短板。

**Token 消耗管理**在 5/20 遇到問題：用 Opus 整頓 schema 一下就燒完 token，顯示在高耗能任務上的資源規劃需要改進。

**鄧白氏漲價談判**：從月初開始一路追蹤到最終破局（5/20），讓先前設計的自動化流程全部報廢，是一個因外部因素導致前期投入歸零的典型情況，心理成本不小（「撒優娜拉的話我就白做了自動化」）。

**跨部門協調的阻力**：IT 部門面子問題、經辦離職導致對口空窗、無法聯絡到對口綠能部長（三顧茅廬）、排程小幫手漏邀通知等行政摩擦耗費不少精力。

## 📚 學習與成長

**Git 與 Claude Code 實戰經驗**：從月初懵懂試探，到月末已能以「監工模式」操作，理解了「目標導向任務用 Claude 比較穩定」的使用哲學，並摸索出控制 AI 工作節奏的技巧。

**Power Platform 生態系認識**：親手評估 Power Apps 與 Power Automate，理解了雲端版與桌面板功能不同、可能需要兩者並用的實際限制，而非只看官方宣傳。

**資料庫與 ER Model 概念實作**：雖然過程磕磕絆絆，但完成了從概念到空表架設的完整流程，對資料流設計有了更具體的認識（「好像有點反我的直覺」反映的正是真實學習的痕跡）。

**績效自我管理意識**（5/20）：學長建議每個月整理自己的績效，以便與主管對齊、讓工作可見。這是一個職涯管理的重要觀念，值得落實。

**系統比對的複雜性**：透過新企金系統與舊系統的比對實作，體驗到「同一系統不同天查出不同資料」的資料一致性問題，學到了在技術決策中重視資料可信度的重要性。

## 💡 其他觀察與建議

**工具切換成本值得正視**：本月同時評估 Power Apps、Power Automate、Claude Code、Cowork、VS Code 等多種工具，且常在多工具間切換操作，造成大量認知負擔。建議下個月聚焦在 2-3 個核心工具，減少工具切換帶來的思緒碎片化。

**AI 使用策略需要更清晰**：Claude 容易「覺得自己完成了」但實際未完成，這個問題在月底仍然存在。建議建立更明確的「任務驗收標準」，在交辦 AI 任務前先定義完成條件，而非邊做邊判斷。

**績效可視化是現在就該做的事**：學長的建議（每月整理績效）非常實用。本月做了相當多事情，但因為分散在各個系統與任務中，若不主動整理，很難讓主管一眼看到全貌。從 6 月開始落實月底績效整理，可作為工作習慣的一部分。

**資料庫技能是急迫補強項**：多次在資料庫操作上卡關（下指令不熟、匯入失敗、資料不一致），而資料庫操作是目前幾個核心專案的基礎技能。建議利用空檔時間系統性補強 SQL 基本操作，特別是匯入/匯出、條件查詢與資料比對。
