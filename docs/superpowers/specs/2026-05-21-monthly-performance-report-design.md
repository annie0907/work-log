# 月績效報告設計文件

**日期：** 2026-05-21
**狀態：** 已核准

---

## 背景與目標

每月從兩個資料來源（git repo 每日記錄 + Notion 任務資料庫）自動彙整出一份給上級主管的正式月績效報告，與現有的個人月摘要（`YYYY-MM-summary.md`）分開存放。

---

## 資料來源

| 來源 | 用途 |
|------|------|
| `YYYY/MM/YYYY-MM-DD.md`（每日記錄） | 主要完成事項細節、挑戰描述、下月待辦 |
| Notion 資料庫：`2fc0cd17-d218-8011-b93d-000b2d424809`（2026年工作紀錄資料庫） | 任務清單、狀態、類型、主題分類、實際完成日期 |

Notion 資料庫欄位：
- `名稱`：任務名稱
- `任務類型`：例行公事 / 專案進度 / 其他事務
- `主題分類`：中華徵信所 / 鄧白氏 / 金腦 / 風管會 / 行政 / 其他
- `狀態`：本週預計完成 / 尚未完成 / 上週已完成 / 不再追蹤
- `日期`：任務日期
- `實際完成日期`：實際完成日期

---

## 輸出檔案

- **路徑：** `YYYY/MM/YYYY-MM-performance.md`
- **語言：** 繁體中文
- **格式：** Markdown

現有的 `YYYY-MM-summary.md` 保持不動，兩者互不影響。

---

## 報告結構

```markdown
# YYYY 年 MM 月 工作績效報告

## 一、執行摘要

2-3 句話，本月整體成效的高階概述，適合主管快速掌握重點。

## 二、主要完成事項

依主題分類（風管會、中華徵信所、鄧白氏、金腦、行政…）列出本月已完成任務，
重點說明「做了什麼、達到什麼結果」。

## 三、專案進度追蹤

各進行中專案（任務類型＝專案進度）的當前狀態、本月推進了什麼、有無阻礙。

## 四、例行業務執行狀況

例行公事類任務的完成情形與品質說明。

## 五、遇到的挑戰與解決方式

從每日記錄中擷取重要障礙及如何克服（省略瑣碎日常）。

## 六、下月工作計畫

根據月底明日待辦 + Notion 尚未完成的任務，列出下月預計重點。
```

## 內容來源對應

| 章節 | 主要來源 |
|------|---------|
| 一、執行摘要 | Claude 綜合兩個來源後撰寫 |
| 二、主要完成事項 | Notion（狀態＝上週已完成，依主題分類） + 每日記錄補充細節 |
| 三、專案進度追蹤 | Notion（任務類型＝專案進度） |
| 四、例行業務執行狀況 | Notion（任務類型＝例行公事） |
| 五、遇到的挑戰與解決方式 | 每日記錄（今日進度中的阻礙描述） |
| 六、下月工作計畫 | 月底每日記錄的明日待辦 + Notion 尚未完成任務 |

## 觸發方式

**初期：手動觸發**
在對話中說「幫我產出 YYYY 年 MM 月的績效報告」，Claude 執行以下步驟：

1. 讀取 `YYYY/MM/` 目錄下所有當月每日記錄
2. 查詢 Notion 資料庫，篩選當月任務（依 `日期` 或 `實際完成日期`）
3. 依報告結構彙整內容
4. 產出 `YYYY/MM/YYYY-MM-performance.md` 並存入 git repo

**未來可選：自動排程**
使用 `schedule` skill 設定每月最後一個工作日自動執行。
